遇见Hexo

  • Hexo
随笔

总感觉相见恨晚

       Hexo是一个开源的静态博客生成器，用node.js开发，作者是台湾大学生tommy351.

初识Hexo

       说一说我是怎么和Hexo认识的吧！说起认识那就不得不说 GitHub 了，github是一个非常之大的开源项目免费托管库。在我接触Java和android之后就发现了这功能强大的GitHub库，随着对github的深入了解和使用，慢慢发现了 GitHub 不仅可以托管开源项目，而且还可以托管静态网页，那么当下如此流行的Hexo自然而然的和我相识了。

为什么激动

1、玩过动态博客的应该都知道主机商通常是“免费的不稳定,稳定的不免费”。
2、阮一峰前辈曾说,blogger分为三个阶段。
       a、最开始,是门户博客。
       b、之后,是独立博客。
       c、最后,觉得独立博客自己管理起来费劲,便找个别人来管 的空间,自己负责写就好。
       d、 GitHub 帮你托管而且管理权还在bloger手中
3、可以轻松的玩转博客，静静的装会b

小小的遗憾

       动态博客更新文章时,脚本是不变的,只需要更新数据库。静态博客要频繁改动文件,不支持增量式上传的东西,比如ftp,就难于管理。此外,还要十分熟悉git各种命令,才能部署页面。毕竟鱼与熊掌不可兼得也，虽然需要频繁的改动文件，但是还是有很多的方便之处。

Hexo和 GitHub 的博客之路待续

       警告：小白入门需谨慎
              1、需要折腾，必须要有耐心。
              2、需要一定的学习能力和钻研精神。
              3、需要懂一些晚些基础知识，如果不懂更需要花时间摸索。
              4、需要考虑好博客建成能否坚持写作，如果不能就不要浪费时间了。
              5、需要问自己三遍坚持写博客很难很难。
       从我决定建立个人博客开始到现在已经3天了，而且还要上课。在课上满脑子都是，怎样使用git, git是什么 ，node.js是什么 ，github是什么 。
